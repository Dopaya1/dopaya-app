## Integration Strategy

### Phase 1: Subdomain (Safe, Recommended Start)

```
brands.dopaya.com/brands/nikin-clean-energy-donation-rewards
```

**How:**
- Deploy Astro project to Vercel (separate project)
- Point `brands.dopaya.com` DNS to Vercel deployment
- Test and monitor

**Pros:**
- Zero risk to main site
- Easy to test and iterate
- Simple to rollback (just remove DNS)

---

### Phase 2: Path-Based (After Successful Testing)

```
dopaya.com/brands/nikin-clean-energy-donation-rewards
```

**How (Option 1 - Rewrites, No Code Changes):**

In your **main Dopaya Vercel project**, add to `vercel.json`:
```json
{
  "rewrites": [
    {
      "source": "/brands/:path*",
      "destination": "https://brands.dopaya.com/brands/:path*"
    }
  ]
}
```

**Pros:**
- Main domain SEO benefit
- No code changes to main app
- Still fully isolated deployments

**How (Option 2 - Monorepo Build):**

More complex but gives single deployment. Only do this if Option 1 doesn't work for you.

## Customization Guide

### Update Brand Information

Edit `src/data/brands.json`:
```json
{
  "id": "nikin",
  "name": "NIKIN",
  "description": "Update this text...",
  ...
}
```

Save, rebuild, redeploy.

### Add New Brand Partner

Add to `src/data/brands.json`:
```json
{
  "id": "new-brand",
  "name": "New Brand Name",
  "slug": "new-brand",
  ...
}
```

This automatically generates 20 new pages (1 brand Ã— 5 sectors Ã— 4 rewards).

- `Hero.astro` - Hero section design
- `HowItWorks.astro` - Process steps

Changes apply to all 160 pages.
